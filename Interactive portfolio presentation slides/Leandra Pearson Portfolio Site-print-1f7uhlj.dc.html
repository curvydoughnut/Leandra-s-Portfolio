Executely

Create a new application center full of AI assisted products for a faster ideation to MVP work flow.

The challenge

Startup teams often struggle to move from idea to execution due to a lack of clear guidance and overwhelming tool choices.

Specific goals

Increase user retention by creating a new feature, creating consistent branding.

Key metrics

Reduced production cycle by 30%. Enhanced customer retention & satisfaction by 25%.

Company
Executely
Role
User Experience Designer
Year
2O24

The Problem

Startup teams struggle to move from idea to execution because existing tools feel fragmented, overly complex, and visually cold. Without clear guidance or curated resources, early stage workflows stall exactly when clarity and speed matter most.

How I Solved It

  1. Slowed down and listened. Studied the real friction users were feeling before reaching for a redesign.
  2. Studied proven dashboard patterns. Built on frameworks people already understood rather than reinventing the wheel.
  3. Expanded from a popup to a full-page system. Immediately made navigation feel less overwhelming.
  4. Leaned into a warmer dark themed interface. Inspired by Apple and Shopify so the platform felt inviting, not intimidating.

The result was a scalable AI Resource Center that met users where they already were, both mentally and visually. By leaning into established patterns and refined aesthetics shaped by products like ChatGPT and modern productivity tools, the platform felt immediately familiar without sacrificing function.

Locci Box

A B2B execution infrastructure platform. Providing engineers with a space to create, test, and play.

The challenge

Build solutions that improve how software is built. Work with IBM BOB, a program that understands your codebase, helping you reduce repetitive work and build with real context.

Specific goals

Create a sandbox in which engineers can write, test and play with their code before deployment.

Key metrics

94ms execution time, 100% tenant isolation, 3× faster review cycles.

Company
IBM
Role
Product Designer
Length
48 Hours
Year
2O26

The Problem

AI agents and enterprise developers were writing and running code with no safe place to execute it. Existing sandbox tools were built for developers who live in the command line, not for teams who need visibility, speed, and security at once. Code and tests shared the same environment, meaning a bug in one could corrupt the other. There was no zero trust runtime, no isolated execution, and no human friendly interface. The tooling simply wasn’t built for the agent era.

How I Solved It

  1. Researched the concept during handoff. Used the wait window to figure out where I could contribute most as the UX/UI designer on a two-person team.
  2. Vibe coded the interface in Lovable. Iterated in real time, moving between the live project and a vibe-coded wireframe to keep each honest.
  3. Mapped four core surfaces under one mental model. Dashboard, Code Editor, BOB AI Reviewer, and API Keys manager, so users never felt lost.
  4. Designed the Dual Sandbox and a 4-step BOB loop. Code and tests run side by side in under 900ms; the reviewer follows submit, analyze, execute, prove, so AI feedback felt verified, not just suggested.

Locci Box shipped as a live B2B SaaS platform built on React, TypeScript, Express, and MCP. Hardware isolated microVMs boot in under 100ms per tenant, paired with a full in browser IDE, an AI code reviewer that runs the fix rather than just suggesting it, and an API key manager with rate limits and audit trail. It can also plug into Claude as a local MCP server, meeting developers where they already work.

Product

Locci Box

Every agent deserves its own computer. Write code. Test code. Ship code — in isolated microVMs.

Meal Prep Pal

A B2C online platform that provides users with meal prepping, step-by-step recipe guidance, combines favorited meals into a digital recipe book, plus more!

The challenge

A synchronized, heads-down build session happening across AI Tinkerers cities worldwide – one global challenge, one submission platform, one day.

Specific goals

Create a platform that is built with generative AI that tackles a real world problem.

Key metrics

8 hour execution time

Company
AI Tinkerers
Role
Product Designer
Length
8 Hours
Year
2O26

The Problem

Most people do not hate cooking. They hate the part that comes before it. The average American makes over 200 food related decisions per day, and by dinnertime decision fatigue has already won. Recipe apps, pantry trackers, nutrition apps, and grocery lists all lived separately, forcing users to carry their own context across four tools every week. No single product connected what someone already had in their kitchen to what they should cook next, and American households were losing over $2,000 a year to food waste because of it.

How I Solved It

  1. Audited the competitive landscape to find where the experience broke. Every tool treated diet, budget, time, pantry inventory, and serving size as separate inputs when users think about them all at once. That gap became the design thesis.
  2. Made the Pantry to Grocery List connection the centerpiece of the product. When the AI knows what you already own, it skips those items on your shopping list automatically. It sounds simple but it is the exact loop every other app in the category fails to close.
  3. Designed the empty state chat prompts to teach users the grammar of the product without any onboarding. "Build a 7-day high-protein plan, around 2,200 calories, no pork, $80 budget" is not filler copy. It shows users the full range of what they can ask for before they type a single word.
  4. Applied a favorites first, A to Z hierarchy to the recipe book and surfaced cook time and calories above the fold on every recipe card, designing for how people scan when they are tired, not how they browse when they have time.
Product

Meal Prep Pal

Every week deserves a better dinner plan. Set your goals. Stock your pantry. Get the week — without the planning.
